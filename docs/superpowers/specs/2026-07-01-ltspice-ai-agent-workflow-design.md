## 1. 背景与目标

### 1.1 使用者画像
- **非常懂需求**，能清楚说出"我要一个截止频率 1kHz 的低通滤波器"这类目标。
- **是电路小白**，不懂拓扑选型、器件计算，不熟练 LTspice 图形操作（拖元件、连线、设参数对其很痛苦）。
- 工作环境具备：**AI agent** + **LTspice 软件** + **Python**。

### 1.2 目标
建立一套 **AI agent 辅助的硬件设计工作流与工具链**，让使用者只需用自然语言提需求，由 agent 驱动 LTspice 完成电路的**设计 → 仿真 → 迭代 → 交付**全流程。

### 1.3 交付定位
本方案需**可交付给他人使用**，因此除工具外，还包含完备的**设计方案、项目工程、工作指引文档**。

## 3. 核心架构

### 3.1 核心思想
**网表（netlist）是"真相源"（source of truth），其余一切都是围绕它的自动化。**

- **网表 `.cir`**：纯文本电路描述，agent 生成极其可靠，LTspice 可直接 `-b` 批处理仿真。
- **原理图 `.asc`**：带坐标的图形文件，由网表**自动布局**生成，供人查看/学习/微调，但**不作为仿真输入的真相源**（避免"agent 直接画坐标"这一最易出错环节）。
- **agent 靠"读指标数字"判断电路好坏并迭代**，不依赖"看波形图"，从而使闭环全自动、可靠。

### 3.2 架构图

```
你（提需求，看结论）
      │  自然语言："做个截止频率1kHz的低通滤波器，看频响"
      ▼
┌─────────────────────────────────────────────┐
│  AI Agent（大脑）                              │
│  · 把需求翻译成电路拓扑 + 器件参数              │
│  · 调用下面的工具，读仿真指标，判断达标没        │
│  · 没达标 → 改参数 → 重跑（自动迭代）           │
└─────────────────────────────────────────────┘
      │ 生成/修改
      ▼
  电路网表 .cir  ◄──── 真相源（纯文本，agent 可靠生成）
      │
      ├──► [工具1] run_sim.py   ── 调 LTspice.exe -b 跑仿真 ──► .raw 波形
      │                                                          │
      │                                          [工具2] parse_raw.py
      │                                          解析波形 → 关键指标(增益/带宽/纹波…)
      │                                                          │
      │                                                          ▼
      │                                              agent 读指标，判断+迭代
      │
      └──► [工具3] net2asc.py  ── 自动布局 ──► 原理图 .asc
                                                  │
                                                  ▼
                                      你在 LTspice 打开，看图/学习/微调
```

### 3.3 为什么这样设计（关键权衡）
- 仿真闭环 100% 走网表 → **可靠、可全自动**。
- 原理图作为网表的"可视化投影" → **既能看图学习，又避开坐标生成的高出错风险**。
- agent 读指标而非读图 → **判断/迭代无需人类介入**。

## 4. 工作流：一次完整的设计迭代

别人拿到这套东西后，日常实际使用流程：

```
① 你说需求        "做个截止1kHz、增益0dB的二阶低通，输入10mV正弦，看频响和纹波"
        ↓
② agent 拆解      选拓扑(Sallen-Key)→查配方库→算元件值→生成 design.cir
        ↓
③ 自动仿真        run_sim.py design.cir → design.raw
        ↓
④ 提取指标        parse_raw.py → {截止频率:1.02kHz, 通带增益:-0.1dB, Q:0.71, ...}
        ↓
⑤ agent 判定      对照需求：截止频率偏高2% → 微调 R 值 → 回到③
        ↓（自动迭代 2-3 轮直到达标）
⑥ 出图给你        net2asc.py → design.asc（可在 LTspice 打开）
        ↓
⑦ 交付报告        report.md：需求→最终电路→BOM→仿真结果→波形图→说明
```

**用户只做 ① 和验收 ⑦**，中间全自动。**每一步留痕**（网表、指标、迭代记录归档），保证可复现、可审查。

## 6. 四个核心工具（职责与接口）

每个工具单一职责、命令行可独立调用、有明确输入输出，便于 agent 组合与独立测试。

### 6.1 `run_sim.py` — 仿真执行器
- **职责**：输入网表，调用 `LTspice.exe -b` 批处理仿真，产出 `.raw` 与 `.log`。
- **接口**：`python tools/run_sim.py <path/to/design.cir>` → 生成同名 `.raw`/`.log`，退出码表示成功/失败。
- **依赖**：`config.yaml` 中的 LTspice 路径。
- **错误处理**：LTspice 报错时解析 `.log`，返回可读错误信息（如"节点悬空""器件模型缺失"）。

### 6.2 `parse_raw.py` — 波形解析器
- **职责**：读取 `.raw`，按分析类型（.ac/.tran/.dc）提取关键指标，输出 `metrics.json`。
- **接口**：`python tools/parse_raw.py <design.raw> [--type ac|tran|dc]` → 打印/写出指标 JSON。
- **提取指标示例**：
  - AC：截止频率(-3dB)、通带增益、相位裕度、Q 值、峰化。
  - TRAN：稳态值、纹波峰峰值、上升时间、过冲、建立时间。
  - DC：工作点电压/电流、静态功耗。
- **关键**：让 agent 读数字判断，无需"看图"。

### 6.3 `net2asc.py` — 网表转原理图
- **职责**：解析网表，自动布局，生成可在 LTspice 打开的 `.asc`。
- **接口**：`python tools/net2asc.py <design.cir>` → 生成同名 `.asc`。
- **范围**：第一版保证简单/中等电路出图可读；复杂电路允许人工微调，不追求完美布局。

### 6.4 `circuit_lib.py` — 电路配方库
- **职责**：内置常用电路"配方"（拓扑模板 + 元件计算公式），作为 agent 拼装的积木。
- **接口**：Python 函数库，被 agent 调用生成网表片段。
- **首批配方**（对应用户选择的电路类型：模拟信号 / 基础 / 专业）：
  - 基础：分压、RC 一阶低通/高通、限流。
  - 模拟信号：Sallen-Key 二阶低通/高通、反相/同相运放放大器、有源带通。
  - 可扩展：文档说明如何加新配方。

## 8. 范围裁剪（YAGNI）

第一版**不做**：
- ❌ 图形界面/网页（命令行 + agent 对话足够）。
- ❌ 复杂电路的完美自动布局（先保证简单/中等电路出图，复杂的允许手动微调）。
- ❌ 器件商品化选型（先用理想值，需要时再对到实际元件系列如 E24）。

## 9. 成功标准

1. 使用者用一句自然语言需求，agent 能在无人工干预下产出**通过仿真验证、达标的网表**。
2. 每个电路项目产出完整留痕：`.cir` / `.raw` / `metrics.json` / `iterations.md` / `report.md`。
3. 生成的 `.asc` 能在 LTspice 中正常打开查看。
4. 一个从没接触过本方案的人，仅凭 `docs/` 文档即可独立跑通第一个电路并理解如何提需求。
5. 换一台机器，只改 `config.yaml` 即可运行。
